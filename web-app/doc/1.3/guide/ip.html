3.1 Files for preservation

Digital data that is intended to be preserved is designated as a "master". These are files whose content is to be stored perpetually. Such data is kept in a way to allow for media migration; and the automatic production of derivatives.

The data can be of any content type: audio, video, image, text, software. Simply put: a file with a name that has one or more bytes.

3.2 Files for presentation

Derivatives and other data are used for presentational purposes. Rather then emitting the master files in the dissemination package, the Content Producer can choose to supply custom data. For example png images to offer a preview image for an audio master file.

The files intended for presentation - or content substitution - is temporal in nature and may be deleted or replaced with different content all the time.

The object repository has services to generate derivative content from the master as well. Wether custom derivatives are supplied or automatically generated, their use is entirely optional.
